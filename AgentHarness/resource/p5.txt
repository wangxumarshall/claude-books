Introduction
Last week, I decided to start a new content series for my Medium account, a series of blog posts to both challenge me and share fresh insights in every article with implementations on a WEEKLY BASIS, if I can keep up with the times. While I anticipate covering areas like large language models, neural networks, AI agents, distributed AI/ML systems, and AI frameworks, I’m open to exploring a broader range of subjects as the series evolves. My aim is to deliver implementation-focused content on a consistent weekly schedule.

Since I have been working with Python for a long time on constructing AI agents and workloads, I have been always wondering how to prepare this experience (but simple) on the Rust programming language side for a while, and frankly, I had never looked at the current ecosystem for the Rust side. First, I did a short ecosystem search and find how far the current ecosystem has come for Rust in AI agents. But instead of using these, I decided to make an AI agent from scratch for the weekly challenge and deploy it to edge (client-side) somehow. I was going to be a little rusty and I’ve been wanting to play with WASM for a long time.

First step in this process was to set up my development environment. I installed the latest version of Rust and familiarized myself quickly with its syntax and features. I also explored WebAssembly (WASM), a powerful IR that allows Rust code to run at near-native speed in web browsers. This would enable me to deploy my AI agent efficiently to edge, so, client-side.

Then, hit the cargo new!
Writing my first Rust Agent

Cargo crate representation
What we need to do here is basically: open a new project, construct the AsyncOpenAI client, create a chain structure by defining async traits and custom structures for the Agents, and then bind the top-level interface to the WASM.

We also need to determine how to conceptually construct an agent. To avoid getting bogged down in the agent setup for now, I decided to design an agent that can understand an article from arXiv, interpret it, gather information from various sources using a search engine, and convert this information into an article based on our prompt.

Setting Up Packages
So, first let’s know the components and crates which we will be use:

AsyncOpenAI:

Rust library for OpenAI platform services with pre-built custom structs.
Crate: https://docs.rs/async-openai/latest/async_openai/all.html
AsyncTrait:

In 2023, The Rust Async Working Group is excited to announce major progress towards goal of enabling the use of async fn in traits.
This crate provides an attribute macro to make async fn in traits work with dyn traits.
We write an #[async_trait] macro on top of traits and trait impls that contain async fn, and then they work.
Crate: https://docs.rs/async-trait/latest/async_trait/all.html
Serde & SerdeJson:

Framework for serializing and deserializing Rust data structures efficiently and generically, and its extension SerdeJson which will be used for the manipulation of JSON object of search data.
Serde usage from the docs:
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize, Debug)]
struct Point {
    x: i32,
    y: i32,
}

fn main() {
    let point = Point { x: 1, y: 2 };

    // Convert the Point to a JSON string.
    let serialized = serde_json::to_string(&point).unwrap();

    // Prints serialized = {"x":1,"y":2}
    println!("serialized = {}", serialized);

    // Convert the JSON string back to a Point.
    let deserialized: Point = serde_json::from_str(&serialized).unwrap();

    // Prints deserialized = Point { x: 1, y: 2 }
    println!("deserialized = {:?}", deserialized);
}
Crates: https://crates.io/crates/serde & https://crates.io/crates/serde_json
reqwest:

Convenient, higher-level HTTP Client for Rust.
We will construct the HTTPClient object for arXiv and Search agent with this library.
wasm-bindgen:

This crate contains the runtime support necessary for wasm-bindgen the attribute and tool. Crates pull in the #[wasm_bindgen] attribute through this crate and this crate also provides JS bindings through the JsValue interface.
We will create bindgins with this, and compile.

src/agents/search_agent.rs

use async_openai::{config::OpenAIConfig, Client as OpenAIClient};
use async_trait::async_trait;
use dotenv::dotenv;
use reqwest::header::{HeaderMap, HeaderValue};
use serde_json::Value;
use std::env;

use crate::agents::agent::{Agent, ApiError};

#[derive(Clone)]
pub struct SearchAgent {
    http_client: reqwest::Client,
    system: Option<String>,
    openai_client: OpenAIClient<OpenAIConfig>,
}

#[async_trait]
impl Agent for SearchAgent {
    fn name(&self) -> String {
        "SearchAgent".to_string()
    }

    fn client(&self) -> OpenAIClient<OpenAIConfig> {
        self.openai_client.clone()
    }

    fn system_message(&self) -> String {
        if let Some(message) = &self.system {
            message.to_owned()
        } else {
            "You are an agent. You will receive a question that may be quite short or does not have much context. 
             Your job is to research the Internet and to return with a high-quality summary to the user, assisted by the provided context.
             The provided context will be in JSON format and contains data about the initial Google results for the website or query.
             Be concise.

        Question:
        ".to_string()
        }
    }

    async fn prompt(&self, input: &str, data: String) -> Result<String, ApiError> {
        let input = format!(
            "{input}

            Provided context:
            {}
            ",
            serde_json::to_string_pretty(&data).map_err(|_| ApiError)?
        );
        let res =
            self.client()
                .chat()
                .create(
                    async_openai::types::CreateChatCompletionRequestArgs::default()
                        .model("gpt-3.5-turbo")
                        .messages(vec![
                        async_openai::types::ChatCompletionRequestMessage::System(
                            async_openai::types::ChatCompletionRequestSystemMessageArgs::default()
                                .content(&self.system_message())
                                .build().map_err(|_| ApiError)?,
                        ),
                        async_openai::types::ChatCompletionRequestMessage::User(
                            async_openai::types::ChatCompletionRequestUserMessageArgs::default()
                                .content(input)
                                .build().map_err(|_| ApiError)?,
                        ),
                    ])
                        .build()
                        .map_err(|_| ApiError)?,
                )
                .await
                .map_err(|_| ApiError)?
                .choices[0]
                .message
                .content
                .clone()
                .ok_or(ApiError)?;

        Ok(res)
    }
}

impl SearchAgent {
    pub fn new() -> Self {
        dotenv().ok();
        let api_key = env::var("OPENAI_API_KEY").expect("OPENAI_API_KEY must be set");
        let serper_api_key = env::var("SERPER_API_KEY").expect("SERPER_API_KEY must be set");

        let config = OpenAIConfig::new().with_api_key(api_key);

        let openai_client = OpenAIClient::with_config(config);

        let mut headers = HeaderMap::new();
        headers.insert(
            "X-API-KEY",
            HeaderValue::from_str(&serper_api_key).expect("Invalid SERPER_API_KEY"),
        );
        headers.insert("Content-Type", "application/json".parse().unwrap());

        let http_client = reqwest::Client::builder()
            .default_headers(headers)
            .build()
            .unwrap();

        Self {
            http_client,
            system: None,
            openai_client,
        }
    }

    pub async fn prepare_data(&self, prompt: &str) -> Result<String, ApiError> {
        let json = serde_json::json!({
            "q": prompt
        });

        let res = self
            .http_client
            .post("https://google.serper.dev/search")
            .json(&json)
            .send()
            .await
            .map_err(|_| ApiError)?;

        let json = res.json::<Value>().await.map_err(|_| ApiError)?;
        Ok(serde_json::to_string_pretty(&json).map_err(|_| ApiError)?)
    }
}

use async_openai::{config::OpenAIConfig, Client as OpenAIClient};
use async_trait::async_trait;
use dotenv::dotenv;
use reqwest::Client;
use std::env;

use crate::agents::agent::{Agent, ApiError};

#[derive(Clone)]
pub struct ArxivAgent {
    http_client: reqwest::Client,
    system: Option<String>,
    openai_client: OpenAIClient<OpenAIConfig>,
}

#[async_trait]
impl Agent for ArxivAgent {
    fn name(&self) -> String {
        "ArxivAgent".to_string()
    }

    fn client(&self) -> OpenAIClient<OpenAIConfig> {
        self.openai_client.clone()
    }

    fn system_message(&self) -> String {
        if let Some(message) = &self.system {
            message.to_owned()
        } else {
            "You are an agent that retrieves and summarizes content from arXiv papers. 
             You will receive an arXiv paper ID or URL. Your job is to fetch the paper's metadata and abstract,
             and provide a concise summary of the paper based on this information.
             Be accurate and focus on the key points of the paper.

             Paper ID or URL:
             ".to_string()
        }
    }

    async fn prompt(&self, input: &str, data: String) -> Result<String, ApiError> {
        let input = format!(
            "{input}

            Paper metadata and abstract:
            {}
            ",
            data
        );
        let res =
            self.client()
                .chat()
                .create(
                    async_openai::types::CreateChatCompletionRequestArgs::default()
                        .model("gpt-3.5-turbo")
                        .messages(vec![
                        async_openai::types::ChatCompletionRequestMessage::System(
                            async_openai::types::ChatCompletionRequestSystemMessageArgs::default()
                                .content(&self.system_message())
                                .build().map_err(|_| ApiError)?,
                        ),
                        async_openai::types::ChatCompletionRequestMessage::User(
                            async_openai::types::ChatCompletionRequestUserMessageArgs::default()
                                .content(input)
                                .build().map_err(|_| ApiError)?,
                        ),
                    ])
                        .build()
                        .map_err(|_| ApiError)?,
                )
                .await
                .map_err(|_| ApiError)?
                .choices[0]
                .message
                .content
                .clone()
                .ok_or(ApiError)?;

        Ok(res)
    }
}

impl ArxivAgent {
    pub fn new() -> Self {
        dotenv().ok();
        let api_key = env::var("OPENAI_API_KEY").expect("OPENAI_API_KEY must be set");

        let config = OpenAIConfig::new().with_api_key(api_key);
        let openai_client = OpenAIClient::with_config(config);

        let http_client = Client::new();

        Self {
            http_client,
            system: None,
            openai_client,
        }
    }

    pub async fn prepare_data(&self, input: &str) -> Result<String, ApiError> {
        let arxiv_id = self.extract_arxiv_id(input)?;
        let api_url = format!("http://export.arxiv.org/api/query?id_list={}", arxiv_id);

        let response = self
            .http_client
            .get(&api_url)
            .send()
            .await
            .map_err(|_| ApiError)?
            .text()
            .await
            .map_err(|_| ApiError)?;

        let paper_info = self.parse_arxiv_response(&response)?;
        Ok(paper_info)
    }

    fn extract_arxiv_id(&self, input: &str) -> Result<String, ApiError> {
        let arxiv_id = if input.contains("arxiv.org") {
            input.split('/').last().unwrap_or("").to_string()
        } else {
            input.to_string()
        };

        if arxiv_id.is_empty() {
            return Err(ApiError);
        }

        Ok(arxiv_id)
    }

    fn parse_arxiv_response(&self, response: &str) -> Result<String, ApiError> {
        let title = self.extract_tag_content(response, "title")?;
        let authors = self.extract_tag_content(response, "author")?;
        let abstract_text = self.extract_tag_content(response, "summary")?;

        let paper_info = format!(
            "Title: {}\nAuthors: {}\nAbstract: {}",
            title, authors, abstract_text
        );

        Ok(paper_info)
    }

    fn extract_tag_content(&self, xml: &str, tag: &str) -> Result<String, ApiError> {
        let start_tag = format!("<{}>", tag);
        let end_tag = format!("</{}>", tag);

        let content = xml
            .split(&start_tag)
            .nth(1)
            .and_then(|s| s.split(&end_tag).next())
            .ok_or(ApiError)?
            .trim()
            .to_string();

        Ok(content)
    }
}

src/agent_invoke.rs

use colored::Colorize;

use crate::agents::agent::{Agent, ApiError};
use crate::agents::arxiv_agent::ArxivAgent;
use crate::agents::search_agent::SearchAgent;
use crate::agents::writer::Writer;

pub struct AgentInvoke {
    arxiv_agent: ArxivAgent,
    writer: Writer,
    search_agent: SearchAgent,
}

impl AgentInvoke {
    pub fn new(arxiv_agent: ArxivAgent, writer: Writer, search_agent: SearchAgent) -> Self {
        Self {
            arxiv_agent,
            writer,
            search_agent,
        }
    }

    pub async fn run(&self, arxiv_id: &str, writer_prompt: &str, search_query: &str) -> Result<String, ApiError> {
        println!(
            "{}",
            "================== ENTERING ASSISTANT CHAIN =================="
                .bold()
                .red()
        );
        println!("{}", format!("arXiv ID: {}\n", arxiv_id).red());

        let arxiv_data = self.arxiv_agent.prepare_data(arxiv_id).await?;

        println!("{}", format!("arXiv Data: {}\n", arxiv_data).red());
        
        let search_data = self.search_agent.prepare_data(search_query).await?;
        
        println!("{}", format!("Search Data: {}\n", search_data).red());

        let combined_data = format!("{}\n{}", arxiv_data, search_data);

        let result = self.writer.prompt(writer_prompt, combined_data).await?;
        Ok(result)
    }
}

src/lib.rs

In Rust, lib.rs is the conventional entry point for a library crate. This file is where the crate's public API is defined. When you’re building a WebAssembly module intended to be used as a library from JavaScript, lib.rs serves as the main entry point for defining the Rust code that will be exposed to JavaScript.

Rust has conventions for the structure of projects:

For binary crates, the main entry point is main.rs.
For library crates, the main entry point is lib.rs.
pub mod agents;
pub mod agent_invoke;

use wasm_bindgen::prelude::*;
use wasm_bindgen_futures::JsFuture;
use crate::agent_invoke::AgentInvoke;
use crate::agents::writer::Writer;
use crate::agents::arxiv_agent::ArxivAgent;
use crate::agents::search_agent::SearchAgent;

#[wasm_bindgen(start)]
pub async fn run_agent(arxiv_id: &str, writer_prompt: &str, search_query: &str) -> Result<JsValue, JsValue> {
    console_error_panic_hook::set_once();

    let arxiv_agent = ArxivAgent::new();
    let writer = Writer::new();
    let search_agent = SearchAgent::new();
    let agent = AgentInvoke::new(arxiv_agent, writer, search_agent);
    let result = agent.run(arxiv_id, writer_prompt, search_query).await;

    match result {
        Ok(result) => Ok(JsValue::from_str(&result)),
        Err(e) => Err(JsValue::from_str(&format!("Error: {}", e))),
    }
}

index.js

import init from './pkg/rustageant.js';

async function main() {
    const wasm_agent = await init("./pkg/rustageant_bg.wasm");

    document.getElementById('run_button').addEventListener('click', async () => {
        const arxivId = document.getElementById('arxiv_id').value;
        const writerPrompt = document.getElementById('writer_prompt').value;
        const searchQuery = document.getElementById('search_query').value;

        try {
            const result = await wasm_agent.run_agent(arxivId, writerPrompt, searchQuery);
            console.log(result);
        } catch (error) {
            console.error(error);
        }
    });
}
